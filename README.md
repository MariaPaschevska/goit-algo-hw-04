## Завдання 1 — Рекурсивне копіювання та сортування файлів

### Запуск

```bash
python task1_file_sorter.py <джерело> [призначення]

# Приклади:
python task1_file_sorter.py ~/Downloads          # призначення: ./dist
python task1_file_sorter.py ~/Downloads ~/sorted
```

### Як працює

1. **Парсинг аргументів** — два аргументи командного рядка: джерело та призначення (за замовчуванням `dist`).
2. **Рекурсивний обхід** — функція `copy_files_recursively()` перебирає всі елементи директорії; якщо знаходить піддиректорію — викликає себе знову.
3. **Копіювання** — кожен файл копіюється до `<призначення>/<розширення>/`. Файли без розширення потрапляють до `no_extension/`. При конфлікті імен до назви додається числовий суфікс.
4. **Обробка виключень** — помилки доступу (`PermissionError`) та інші `OSError` перехоплюються та виводяться без зупинки програми.

## Завдання 2 — Фрактал «Сніжинка Коха»

### Запуск

```bash
python task2_koch_snowflake.py             # рівень 4 (за замовчуванням)
python task2_koch_snowflake.py --level 3   # рівень 3
python task2_koch_snowflake.py --level 6   # рівень 6 (детальніший, але повільніший)
```

### Як працює

Функція `koch_curve(t, order, size)` рекурсивно будує криву Коха:

- **Базовий випадок** (`order == 0`): малює відрізок довжиною `size`.
- **Рекурсивний крок**: ділить відрізок на три рівні частини, замінює середню трикутним виступом — чотири рекурсивні виклики зі `size / 3` та поворотами на ±60°.

Функція `draw_snowflake()` викликає `koch_curve` тричі з поворотом на 120° — утворюючи замкнену сніжинку.

## Завдання 3 — Порівняння алгоритмів сортування

### Запуск

```bash
python task3_sorting_comparison.py
```

### Алгоритми

| Алгоритм | Складність (середня) | Складність (гірший) | Пам'ять |
|---|---|---|---|
| Merge Sort | O(n log n) | O(n log n) | O(n) |
| Insertion Sort | O(n²) | O(n²) | O(1) |
| Timsort | O(n log n) | O(n log n) | O(n) |

### Емпіричні результати

| Набір даних | Merge Sort (мс) | Insertion Sort (мс) | Timsort (мс) |
| --- | --- | --- | --- |
| Малий масив (100) | 0.110 мс | 0.095 мс | 0.006 мс |
| Середній масив (1 000) | 1.204 мс | 12.547 мс | 0.088 мс |
| Великий масив (10 000) | 15.778 мс | 1382.956 мс | 1.471 мс |
| Майже відсортований (10 000) | 13.875 мс | 164.657 мс | 0.467 мс |
| Зворотний порядок (10 000) | 10.281 мс | 2679.722 мс | 0.091 мс |

### Висновки

#### 1. Сортування вставками (Insertion Sort) — O(n²)

Працює прийнятно лише на **малих** масивах (до ~200 елементів), де переваги спрощеної логіки переважають над квадратичним зростанням. На масиві з 10 000 елементів у **зворотному** порядку час виконання склав **~2680 мс** — приблизно у **29 500 разів** повільніше за Timsort. Це наочно підтверджує теоретичну оцінку O(n²).

#### 2. Сортування злиттям (Merge Sort) — O(n log n)

Стабільно масштабується та виконує обіцяні O(n log n) операцій у **будь-якому** випадку. Особливо добре поводиться на зворотних масивах (~10 мс проти ~2680 мс у Insertion Sort). Недолік — витрати O(n) додаткової пам'яті та рекурсивні накладні витрати (call stack).

#### 3. Timsort — гібридний O(n log n)

Timsort є беззаперечним переможцем у всіх тестах:

- **На випадкових великих масивах** — у **~10,7 разів** швидший за Merge Sort і у **~940 разів** швидший за Insertion Sort.
- **На майже відсортованих даних** — у **~29,7 разів** швидший за Merge Sort завдяки виявленню вже наявних «пробігів» (runs).
- **На зворотних масивах** — у **~113 разів** швидший за Merge Sort, оскільки Timsort розпізнає спадний порядок як окремий run і відразу обертає його — всього за O(n).

#### Чому Timsort такий ефективний?

Timsort поєднує сильні сторони обох алгоритмів:

1. **Insertion Sort** — оптимальний для маленьких підмасивів (≤ 64 елементи): немає рекурсії, чудова локальність кешу, константні накладні витрати.
2. **Merge Sort** — гарантований O(n log n) на великих підмасивах.
3. **Виявлення runs** — Timsort сканує вхід на природно відсортовані послідовності й зливає їх, а не сортує заново. Для реальних даних (часткові сортування, монотонні послідовності) це дає суттєве прискорення.
4. **Реалізація у C** — вбудований `sorted()` реалізований нативним кодом CPython, що усуває накладні витрати інтерпретатора Python.

#### Загальний висновок

> Використовуйте вбудовані `sorted()` / `list.sort()` для **будь-яких** практичних задач. Написання власного алгоритму сортування виправдане лише у вузьких випадках: специфічна мова програмування без стандартної бібліотеки, нестандартний критерій порівняння або навчальна мета. В усіх інших ситуаціях Timsort — це оптимальне рішення, підтверджене як теоретично, так і емпірично.
